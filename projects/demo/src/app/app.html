How it works

Simple idea, reliable pattern.

1

Declare

Wrap a standalone component with inputs and a root context.

2

Call

Use Confirm.call() and receive a promise immediately.

3

Render

The host displays the component and tracks active call state.

4

Resolve

The component ends the call with the value the original flow awaits.

State vs. Await

Awaitable UI replaces scattered open flags with one straight-line flow.

  • More code
  • Scattered updates
  • Harder to test
  • Manual cleanup
let open = signal(false);
let answer = signal<boolean | null>(null);

open.set(true);

effect(() => {
  if (answer() !== null) {
    // Continue...
  }
});
const answer = await Confirm.call({
  message: 'Proceed?'
});

if (answer) {
  // Continue...
}
  • Less code
  • Straight-line flow
  • Easier to test
  • No manual cleanup
